# 성원플라워 대시보드 배포 (Vercel + GitHub)

**최종 자료 업데이트:** 어제 회사에서 수정·배포 완료.

Supabase는 이미 사용 중이므로, **GitHub에 올리고 Vercel로 배포**하면 회사 PC/폰에서도 접속해서 쓸 수 있습니다.

---

## 1. GitHub에 올리기

1. GitHub에서 새 저장소 만들기 (예: `seongwon-flower-dashboard`)
2. 터미널에서 프로젝트 폴더로 이동 후:

```bash
cd c:\성원플라워
git init
git add .
git commit -m "초기 커밋"
git branch -M main
git remote add origin https://github.com/본인아이디/저장소이름.git
git push -u origin main
```

- **주의:** `dashboard/.env`는 절대 올리지 마세요 (Supabase 키 노출). `.gitignore`에 `.env`가 있는지 확인하세요.

---

## 2. Vercel에서 배포하기

1. [vercel.com](https://vercel.com) 가입 후 로그인
2. **Add New… → Project** 클릭
3. **Import Git Repository**에서 방금 만든 GitHub 저장소 선택
4. **Configure Project**에서:
   - **Root Directory:** `dashboard` 로 지정 (한 단계만 들어가기)
   - **Framework Preset:** Vite (자동 인식될 수 있음)
   - **Build Command:** `npm run build` (기본값)
   - **Output Directory:** `dist` (기본값)
5. **Environment Variables**에 추가:
   - `VITE_SUPABASE_URL` = Supabase 프로젝트 URL
   - `VITE_SUPABASE_ANON_KEY` = Supabase anon public key  
   (지금 `dashboard/.env`에 있는 값 그대로 넣으면 됩니다)
6. **Deploy** 클릭

배포가 끝나면 `https://프로젝트이름.vercel.app` 같은 주소가 생깁니다.

---

## 3. 회사에서 사용하기

- 생성된 Vercel URL을 **회사 PC/폰 브라우저 주소창**에 넣으면 됩니다.
- Supabase는 이미 인터넷에 있으므로, 어디서든 같은 URL로 접속하면 같은 데이터를 봅니다.
- 코드 수정 후 GitHub에 `git push` 하면 Vercel이 자동으로 다시 배포합니다.

---

---

## 5. 문제 해결

- **빌드 실패:** Vercel 대시보드의 Deployments → 해당 배포 → 로그에서 에러 확인. 대부분 `dashboard` 루트 지정 누락이나 env 미설정입니다.
- **화면은 나오는데 데이터 안 나옴:** Supabase URL/키가 Vercel 환경 변수에 정확히 들어갔는지, Supabase 대시보드에서 anon 키가 활성화돼 있는지 확인하세요.
