# 노랑풍선 같은 파일만 나올 때 — 근본 원인과 해결

## 원인
받는 파일명이 **`2026년_경조화환발주내역_..._(13).xlsx`** 처럼 **(숫자)** 가 붙으면  
→ **예전 코드**가 돌고 있는 겁니다.  
새 코드가 적용됐으면 파일명은 **`경조화환발주내역_..._1739123456789.xlsx`** 처럼 **끝에 긴 숫자**가 붙습니다.

즉, **수정한 코드가 배포·반영되지 않은 상태**에서 다운로드를 누르고 있는 것이 문제입니다.

---

## 해결 순서 (한 번만 따라 하세요)

### 1. 터미널에서 배포
```powershell
cd c:\성원플라워\dashboard
git add -A
git status
```
→ `App.tsx` 등 수정된 파일이 나와야 함.

```powershell
git commit -m "노랑풍선 다운로드 새 파일명 및 생성시각"
git push
```

### 2. Vercel에서 빌드 확인
- https://vercel.com 로그인
- **sungwon-flower** 프로젝트 클릭
- **Deployments** 탭에서 **맨 위 배포가 "Ready"** 인지 확인
- 실패(Error)면 빌드 로그에서 원인 확인

### 3. 브라우저에서 캐시 제거 후 접속
- **시크릿/프라이빗 창**에서 배포된 사이트 주소 접속  
  또는
- 일반 창에서 **Ctrl+Shift+R** (강력 새로고침)

### 4. 다운로드로 확인
- 거래처 **노랑풍선** 선택
- **노랑풍선 엑셀** 선택
- **다운로드** 클릭
- 받은 파일 이름 확인:
  - **끝에 긴 숫자** (예: `_1739123456789.xlsx`) → **새 코드 적용됨**
  - **`(1)` `(13)` 같은 것**만 붙음 → **아직 예전 코드** → 2번·3번 다시 확인

### 5. 엑셀 B1 셀 확인
- 새 코드면 B1에 **"성원플라워 채움 (N건) 생성 2026. 2. 21. 오후 2:35:00"** 처럼 **생성 시각**이 보입니다.
- 다운로드할 때마다 시각이 바뀌면 **매번 새로 만든 파일**이 맞습니다.

---

## 정리
- **같은 파일만 15번 나온다** = 그 사이트가 **예전 JS**를 쓰고 있다는 뜻입니다.
- **push → Vercel 빌드 성공 → 시크릿 창 또는 강력 새로고침 → 다시 다운로드** 순서로 하면 새 코드가 적용됩니다.
