# 거래명세표 인쇄 시 컬럼 폭 조정 — 작업 문서

> 이 문서를 주고 "이거 보면서 작업해줘" 하면 됨.  
> 대상: **일반양식**(하나투어비즈니스 등) + **엔타스형** 명세표 HTML. 인쇄 시 2페이지 나오거나 줄바꿈이 많은 문제 수정.

---

## 1. 수정할 파일

- **`src/App.tsx`** (dashboard 폴더 기준)
- **함수:** `buildStatementHtml()` — 이 함수 안의 **`<style>` 블록**에서 테이블 컬럼 `width` 값만 수정하면 됨.

---

## 2. 일반양식 컬럼 폭 (하나투어비즈니스 등)

화면에서는 한 페이지처럼 보이는데 **인쇄하면 2페이지**로 나가는 문제 해결용.

| 컬럼 (class) | 변경 전 | 변경 후 |
|--------------|--------|--------|
| `.col-item` (품목) | 9% | **11%** |
| `.col-requester` (요청부서) | 14% | **20%** |
| `.col-location` (발송 장소) | 27% | **20%** |
| `.col-qty` (수량) | 8% | **5%** |

→ 품목은 글자 1개 더 들어가게 넓히고, 요청부서와 발송 장소는 **같은 폭(20%)**으로 맞추고, 수량은 줄임.

---

## 3. 엔타스형 컬럼 폭 (엔타스, 엔타스프레쉬미트 등)

엔타스형은 **인쇄 시 품목·요청인·발송 장소**가 여러 줄로 줄바꿈되는 문제.  
`.entas-sheet` 안에서만 아래처럼 적용.

| 컬럼 (class) | 변경 전 | 변경 후 |
|--------------|--------|--------|
| `.entas-sheet .col-month` | (기본 6%) | **5%** |
| `.entas-sheet .col-day` | (기본 6%) | **5%** |
| `.entas-sheet .col-item` | 15% | **20%** |
| `.entas-sheet .col-requester` | 20% | **21%** |
| `.entas-sheet .col-location` | 20% | **21%** |
| `.entas-sheet .col-qty` | (기본 5%) | **4%** |
| `.entas-sheet .col-supply` | (기본 11%) | **9%** |
| `.entas-sheet .col-tax` | (기본 6%) | **5%** |
| `.entas-sheet .col-amount` | (기본 12%) | **11%** |

→ 품목·요청인·발송 장소를 넓히고, 월/일/수량/공급가액/세액/금액을 살짝 줄여서 전체 합 100% 맞추고, 인쇄 시 줄바꿈이 줄어들게 함.

---

## 4. 코드에서 찾을 위치

`App.tsx`에서 `buildStatementHtml` 검색 → 그 안의 `<style>` 블록.

- **일반 컬럼:**  
  `.col-month`, `.col-day`, `.col-item`, `.col-requester`, `.col-location`, `.col-qty` 등이 연달아 나오는 부분.
- **엔타스형:**  
  `.entas-sheet .col-item`, `.entas-sheet .col-requester`, `.entas-sheet .col-location` 등이 있는 부분.

위 표의 **변경 후** 숫자로 바꿔 주면 됨.

---

## 5. 확인 방법

1. 대시보드에서 **명세/내역서 내보내기** → **일반양식** 또는 **엔타스형 명세표** 선택 후 HTML 다운로드.
2. 다운로드한 `.html` 파일을 브라우저로 연 뒤 **인쇄(Ctrl+P)** → 미리보기에서 **한 페이지**에 들어가는지, **줄바꿈**이 줄었는지 확인.
